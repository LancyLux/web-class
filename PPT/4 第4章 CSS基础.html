4.1 CSS 简介 (Theory)

什么是 CSS？

  • 全称：Cascading Style Sheets（层叠样式表）
  • 如果说 HTML 是网页的骨架（负责结构）。
  • 那么 CSS 就是网页的衣服和妆容（负责好看）。
3 / 35

4.1 核心理念：结构与表现分离

老外设计网页时，强调一个原则：

买衣服不要和长骨头混在一起！

  • HTML 只负责告诉浏览器：“这是一个标题”。
  • CSS 负责告诉浏览器：“这个标题是红色的、30像素大”。
  • 好处：换一套 CSS，整个网站就能瞬间换个“皮肤”。

4.2.1 选择器：精准制导的导弹

想要给网页“化妆”，你面临的第一个问题是：

你要给哪一个部位化？

  • 是给所有的段落涂色？
  • 还是只给那个特定的按钮加边框？
CSS 选择器（Selector），就是帮你找到目标元素的工具！

4.2.1 CSS 基本语法公式

记住这个万能句型：谁 { 变成什么样 }

选择器 {
    属性名: 属性值;
    属性名: 属性值;
}
  • 选择器： 目标对象。
  • 大括号： 声明块，里面的内容就是魔法。
  • 分号： 每一句话说完，必须加分号（新手最容易忘！）

4.2.2 标签选择器 (Tag Selector)

概念：直接使用 HTML 的标签名作为选择器。

生活比喻：“范围攻击”。校长在广播里喊：“所有穿校服的同学，到操场集合！”

  • 作用：一次性选中页面上所有的该标签元素。
  • 特点：威力巨大，但不够精准。

任务目标：让页面上所有的段落（p标签）文字变成蓝色。

请在编辑器中输入以下代码：

/* CSS 代码 */

任务 1：预期效果

你的 HTML 如果长这样：

<p>我是段落一</p>
<p>我是段落二</p>
<div>我是一个div</div>
  • 段落一：变成蓝色
  • 段落二：变成蓝色
  • div：保持原来的黑色（因为校长没喊 div）。

4.2.3 ID 选择器 (ID Selector)

概念：为特定的元素设置一个唯一的标识，用 # 来选中它。

生活比喻：“身份证号”。警察说：“身份证号是 110105... 的人出列！”

  • 前提：HTML 标签必须有 id="xxx" 属性。
  • 规则：CSS 中必须用 # 开头。
  • 禁忌：一个页面中，同一个 ID 名称只能出现一次！

动手任务 2：ID 选择器

任务目标：只让“特权VIP”变成红色。

HTML 准备：<p id="vip">我是唯一的 VIP</p>

/* CSS 代码 */
#vip {
    color: red;
    font-size: 40px;
}
注意看：前面有一个井号 (#) 哦！

任务 2：预期效果

经过刚才的代码施法：

  • 页面上那个带有 id="vip" 的段落，瞬间变成了：
  • 我是唯一的 VIP
  • 其他普通的段落依然不受影响。

这就是“精准狙击”的威力！

4.2.4 Class 选择器 (类选择器)

概念：给一类元素贴上相同的标签，用 . （点）来选中它们。

生活比喻：“工作服”。店长说：“所有穿红色服务员制服的，去端盘子！”

  • 前提：HTML 标签有 class="xxx" 属性。
  • 规则：CSS 中必须用 . （英文句号）开头。
  • 优势：可无限次重复使用，最常用的选择器！

动手任务 3：Class 选择器

任务目标：把第一和第三个元素背景变成黄色。

HTML: <div class="box"></div> (准备几个这样的div)

/* CSS 代码 */
.box {
    background-color: yellow;
}
考考你：在 CSS 里，ID 用什么符号？Class 用什么符号？
（答：ID 用 #，Class 用 . ）不要搞混啦！

任务 3：预期效果

只要你的 HTML 标签里写了 class="box"：

  • 不管是 <p> 还是 <h1> 还是 <div>。
  • 它们的背景都会齐刷刷地变成黄色！
  • 这就是 Class 的魅力：高度复用。

4.2.5 群组选择器（打包操作）

如果 h1, h2, p 都需要变成灰色，难道要写三遍吗？太累了！

语法：用逗号 ( , ) 隔开。

/* 群组选择器：组队打怪 */
h1, h2, p {
    color: gray;
}

效果：这三个标签被一起选中，统一修改颜色。省时省力！

16 / 35

4.2.5 后代选择器（精确锁定）

怎么选中“那个特殊盒子里面的段落”，而不影响外面的段落？

语法：用空格 ( ) 隔开。

/* 后代选择器：子子孙孙 */
.box p {
    color: green;
}

翻译：在 class 为 box 的元素内部，寻找所有的 p 标签。

4.3 常用 CSS 属性 (Property)

学会了“找目标”，接下来学习几招“魔法技能”：

  • color: red; - 设置文字颜色（字色）。
  • font-size: 20px; - 设置字体大小。
  • font-weight: bold; - 让文字加粗。
  • text-align: center; - 让文本居中对齐。

4.3 常用 CSS 属性 (续)

除了文字，我们还能改变盒子的外观：

  • background-color: pink; - 给背景涂上粉色。
  • width: 200px; - 设置元素的宽度。
  • height: 100px; - 设置元素的高度。

4.4 CSS 伪类 (Pseudo-classes)

网页不应该是死板的，它应该能感知到用户的操作！

核心伪类：:hover (悬停状态)

生活比喻：“含羞草”。平时是个普通的草，你鼠标“摸”它一下，它就闭合了（变色了）。

  • 作用：当鼠标指针停留在元素上时，触发新的样式。
21 / 35

动手任务 5：制作一个会变色的按钮

任务目标：鼠标放上去，按钮背景变黑，文字变白。

/* 1. 按钮的正常状态 */
.btn {
    background-color: lightgray;
    color: black;
}

/* 2. 鼠标放上去的“激动状态” */
.btn:hover {
    background-color: black;
    color: white;
}

任务 5：预期效果

这可能是你写出的第一个具有“交互感”的网页效果！

  • 没碰它时：它是一个灰色不起眼的按钮。
  • 鼠标滑过：瞬间变成酷炫的黑底白字。
  • 鼠标移开：它又乖乖变回了灰色。
思考：很多网站的链接鼠标放上去会下划线消失，就是用 a:hover 做的！

4.5 CSS 代码写在哪里？

目前为止，我们只讲了 CSS 代码长什么样。

但是，这串魔法代码，到底应该写在 HTML 文件的哪个位置呢？

在前端开发中，我们有 三种 穿衣服的方式：

  1. 内嵌样式（Inline）
  2. 内部样式（Internal）
  3. 外部样式（External）
24 / 35

4.5.1 内嵌样式（直接纹身）

做法：直接在 HTML 标签的 style 属性里写 CSS。

<h1 style="color: red; font-size: 20px;">
    我是红色的标题
</h1>
  • 优点：最直接，优先级极高。
  • 缺点：就像在皮肤上纹身，HTML和CSS完全混在一起，后期极难维护！老师强烈不推荐！
25 / 35

4.5.2 内部样式（储物间）

做法：写在 HTML 文件的 <head> 标签里的 <style> 中。 

<head>
    <style>
        h1 { color: red; }
    </style>
</head>
  • 优点：单个页面内做到了结构和表现分离。
  • 缺点：如果你的网站有 100 个页面，你得复制 100 遍这段代码。
26 / 35

4.5.3 外部样式（独立衣柜）

做法：创建一个独立的 .css 文件，然后在 HTML 中“引入”它。

<!-- 在 HTML 的 head 中引入 -->
<link rel="stylesheet" href="style.css">
  • 这是业界最标准、最专业的做法！
  • 所有的样式都保存在外部的 style.css 中。
  • 一次修改，100个页面同时生效！
27 / 35

4.5 样式位置终极总结

方式 代码位置 适用场景 老师推荐度
内嵌 (Inline) HTML 标签内 临时测试、极个别特殊样式 ⭐ (尽量别用)
内部 (Internal) head的style里 单页面宣传页、初学练习 ⭐⭐⭐
外部 (External) 独立的 .css 文件 企业级正规大型项目 ⭐⭐⭐⭐⭐

4.6 层叠、继承与优先级

如果有两段 CSS 代码，都试图改变同一个元素的颜色，浏览器该听谁的？

这就好比：妈妈让你穿红衣服，爸爸让你穿蓝衣服，你穿啥？

这就引出了 CSS 的三大底层逻辑：
层叠 (Cascading)、继承 (Inheritance)、优先级 (Specificity)。
29 / 35

4.6.1 层叠 (Cascading)

原则：“后来居上”。

当多个相同优先级的规则应用在同一个元素时，写在后面的代码会覆盖前面的代码。 

p { color: red; }

/* 因为这行在后面，所以最终段落是蓝色！ */
p { color: blue; }

这就是 Cascading Style Sheets 中 Cascading 的字面含义！

30 / 35

4.6.2 继承 (Inheritance)

原则：“父传子”。某些属性，祖先元素设置了，后代元素会自动享有。

body {
    font-size: 20px;
    color: #333;
}
  • 你在 body 上设置了字体颜色，里面的 p, div 都会自动变成这个颜色。
  • 注意：并不是所有属性都会继承！像边框(border)、背景(background)就不会继承（你爸的房子不一定直接变成你的房子）。
31 / 35

4.6.3 优先级 (Specificity) - 权重之战

如果选择器不同，浏览器会计算它们的“权重”。谁的官大，听谁的。

权重排行榜：

  1. 内嵌样式表 (style="...") 👉 犹如圣旨 (权重 1000)
  2. ID 选择器 (#xxx) 👉 犹如省长 (权重 100)
  3. Class 选择器 (.xxx) 👉 犹如市长 (权重 10)
  4. 标签选择器 (p, div) 👉 犹如村长 (权重 1)
32 / 35

动手任务 6：权重算数题

任务目标：判断文字最终是什么颜色？

/* CSS */
#box { color: red; }   /* 权重 100 */
.text { color: blue; } /* 权重 10 */
p { color: green; }    /* 权重 1 */

<!-- HTML -->
<p id="box" class="text">我是什么颜色？</p>
答案揭晓：是红色！因为 ID 的权重 (100) 碾压了 Class 和标签。

总结：你今天学到了什么？

  • 🎨 CSS 的作用： HTML的化妆师，分离结构与表现。
  • 🎯 寻找目标： 标签(村长)、Class(市长, 最常用)、ID(省长, 唯一)。
  • 💅 魔法属性： color, background, font-size...
  • 💡 交互伪类： :hover 鼠标悬停特效。
  • 📁 引入位置： 强烈推荐使用外部引入（<link>）。
  • ⚖️ 底层规则： 层叠（后来居上）、继承（父传子）、优先级（权重计算）。
