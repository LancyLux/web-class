7.1 基于HTML5的前端开发

生活化比喻：

以前盖房子（HTML4），大家都用通用的砖头 <div>。虽然能盖起来，但没人知道哪块是门，哪块是窗。

HTML5 就像是提供了定制化的建材：有了专门做导航的 <nav>，专门做文章的 <article>，专门做底部的 <footer>。

💻 观察现代 HTML5 骨架：

<body>
  <header>这是网页头部</header>
  <nav>菜单1 | 菜单2</nav>
  <main>
    <article>核心文章内容</article>
  </main>
  <footer>底部版权信息</footer>
</body>

优势：

  • 语义化： 结构清晰，不仅人看得懂，搜索引擎（百度、谷歌）也看得懂，利于 SEO 排名。
  • 自带魔法： 不需要复杂的插件就能播放视频 <video>、音频 <audio>。

7.2 前端开发规范：为什么需要规范？

想象一下，如果没有交通规则，马路会变成什么样？碰碰车现场！

同理，如果没有代码规范：

  • 你找不到图片放在了哪里...
  • 别人的 CSS 把你的页面样式覆盖了...
  • 交接代码时，同事拿着刀站在你背后...

规范，就是开发者的“护身符”！

7.2.1 目录结构 & HTML 规范

📂 目录结构（像整理衣柜）

my-project/
├── index.html   (入口页面)
├── css/         (专放样式)
│   └── style.css
├── js/          (专放脚本)
│   └── main.js
└── images/      (专放图片)
    └── logo.png

✍️ HTML 规范（像写作文）

  • 小写原则： 标签和属性名必须小写。<DIV> 是违法的！
  • 闭合原则： 有始有终 <p>段落</p> 或 <img />。
  • 属性引号： 属性值必加双引号 class="title"。
  • 层级缩进： 父子关系必须用 Tab 键缩进对齐。

7.3 CSS 常用属性 (核心重点)

🌟 准备好你的“网页美颜相机” 🌟

接下来的内容非常酷炫，全程【左边代码，右边效果】，请不要眨眼！

7.3.1 背景 (Background) 理论详解

生活化比喻： 给你的空房子刷漆、贴壁纸。

  • background-color: 背景颜色（刷漆）。
  • background-image: 背景图（贴壁纸）。
  • background-repeat: 图片是否平铺重复 (常用 no-repeat)。
  • background-position (重点)：图片贴在墙上的坐标！
    • 文字方向： center center(正中间), left top(左上角)。
    • 具体像素(X轴 Y轴)： 50px 100px 意思是“图片距离左边 50像素，距离顶部 100像素”。
    • 百分比： 50% 50% 相当于绝对居中。

7.3.1 任务：刷漆与精确贴壁纸

💻 简写魔法 (推荐)：

语法：颜色 图片 平铺 位置

.box {
  width: 100%; height: 200px;
  
  /* 浅蓝色底，不平铺，使用像素精确控制位置 */
  background: lightblue 
      url('star.png') 
      no-repeat 
      30px 50px; /* X偏30, Y偏50 */
}
⭐

X偏移30px，Y偏移50px

7.3.2 无序列表前缀定制

干掉列表默认的“丑黑点”，换成我们自己的图片或小图标。

💻 定制新闻列表：

<ul class="news-list">
  <li>重磅！前端老师太帅被围观</li>
  <li>震惊！网页美翻了</li>
</ul>

<style>
.news-list {
  list-style-type: none; /* 干掉黑点 */
}
.news-list li {
  /* 利用刚刚学的背景坐标控制图标 */
  background: url('hot.png') no-repeat left center;
  padding-left: 30px; /* 留出空隙 */
}
</style>
  • 🔥重磅！前端老师太帅被围观
  • 🔥震惊！用了这个属性网页美翻了

7.3.3 CSS3 背景缩放 (background-size)

图片太大，盒子太小？CSS3 救世主属性解决比例问题！

💻 常见缩放模式：

.bg-cover {
  /* cover: 保证覆盖铺满盒子，可能被裁切 */
  background-size: cover; 
}

.bg-contain {
  /* contain: 保证图片完整显示，盒子可能留白 */
  background-size: contain;
}
cover 撑满全盒
contain
显示完整

7.3.4 渐变背景 (Gradient)

不需要求设计师切图了，用代码直接画出顺滑的颜色过渡层！

💻 线性渐变：

.gradient-box {
  width: 100%;
  height: 200px;
  
  /* 语法：方向, 颜色1, 颜色2... */
  background: linear-gradient(
    to bottom right, /* 从左上到右下 */
    #ff7eb3,         /* 少女粉 */
    #ff758c          /* 橘红色 */
  );
  
  border-radius: 10px;
}
绝美朝霞渐变

7.3.5 核心文字属性 (Typography)

排版好坏直接决定网页气质。拯救密密麻麻的阅读体验！

💻 舒适阅读排版：

.article {
  /* 别用纯黑，深灰更护眼 */
  color: #333333; 
  /* 适合阅读的字号 */
  font-size: 18px; 
  font-weight: bold; /* 加粗 */
  /* 行高设为字号的1.6倍，拒绝拥挤 */
  line-height: 1.6; 
  text-align: justify; /* 两端对齐 */
  text-decoration: underline; /* 下划线 */
}

代码不仅仅是写给机器看的，更是写给未来的自己和同事看的。优雅的文字排版，行高设定为 1.6 倍后，文字有了呼吸的空间，阅读起来犹如清风拂面，极其舒适。

7.3.6 圆角 (Border-Radius)

直角太锐利，圆角更亲和。一键让方块变圆润。

💻 制作圆角与头像：

/* 1. 轻微圆角 (常用于卡片/按钮) */
.btn {
  border-radius: 10px; 
}

/* 2. 完美的正圆形 (常用于头像) */
/* 必须保证 width 和 height 相同 */
.avatar {
  width: 120px; height: 120px;
  border-radius: 50%; /* 核心魔法 */
  border: 4px solid white;
}
👨‍💻

7.3.7 阴影 (Box-Shadow) 🌟

想让元素有“悬浮”在页面的三维立体感？这就是 3D 魔法棒！

💻 语法与悬浮效果：

语法: X偏移 Y偏移 模糊 颜色

.card {
  /* 0偏移, 下沉10px, 模糊20px, 15%黑 */
  box-shadow: 0 10px 20px rgba(0,0,0,0.15);
  transition: transform 0.3s;
}

/* 鼠标放上去时往上飘，阴影加深 */
.card:hover {
  transform: translateY(-10px);
  box-shadow: 0 15px 30px rgba(0,0,0,0.25);
}

7.3.8 边框背景 (Border-Image)

产品经理想要相框一样复杂的花纹边框？把一张图“切割”后铺满边框！

💻 边框图片/渐变：

.fancy-border {
  /* 必须先给边框占位宽度！ */
  border: 10px solid transparent; 
  
  /* 真实项目中引入花纹图片: 
   border-image: url('xxx.png') 30 round; 
  */
  
  /* 这里教你一个黑科技：直接用渐变做边框图！ */
  border-image: linear-gradient(
    to right, red, yellow
  ) 1 stretch;
}
炫酷渐变边框

7.3.9 CSS3多列排版 (Multi-column)

还记得看报纸的排版吗？以前需要算宽度、用 JS，现在只需两行代码！

💻 报纸风格：

.newspaper {
  /* 强制分成 3 列 */
  column-count: 3;
  
  /* 列与列之间留出 20px 空隙 */
  column-gap: 20px;
  
  /* 列与列之间加一条虚线分割 */
  column-rule: 2px dashed #95a5a6; 
}
网页不是一块铁板，它是由小零件拼装起来的乐高。好的设计必须遵循一致性、易读性和反馈性。文字会自动流向下一列，完全不需要我们手动去切断文章。是不是极其智能？

7.3.10 HTML元素隐藏 (面试必考)

怎么让元素看不见？有两种截然不同的魔法：

💻 物理消灭 vs 光学隐身：

.vanish {
  /* 灭霸打响指：连带位置一起抹除 */
  display: none; 
}

.invisible {
  /* 隐形斗篷：看不见了，但坑还占着 */
  visibility: hidden; 
}
盒子A 盒子B (display: none) 盒子C (B的位置没了)
盒子A 盒子B (visibility) 盒子C (B坑还在)

7.4 网页元素设计综合运用

🎨 设计不是玄学，是逻辑与美的结合

三大原则：一致性 (长得一样) 、 易读性 (对比度高) 、 反馈性 (悬浮有动画)。

🔥 终极任务：设计一个具有灵魂的按钮

结合今天学过的背景渐变、圆角、阴影、文字样式，打造现代“行动召唤按钮”。

.perfect-btn {
  padding: 12px 30px; /* 撑起空间 */
  background: linear-gradient(
    to right, #6dd5ed, #2193b0
  ); 
  color: white; border: none;
  border-radius: 30px; /* 胶囊圆角 */
  box-shadow: 0 4px 15px rgba(33, 147, 176, 0.4);
  transition: all 0.3s ease; /* 动画过渡 */
  cursor: pointer; font-size: 20px;
}

.perfect-btn:hover {
  transform: translateY(-3px); /* 悬浮跳动 */
  box-shadow: 0 8px 25px rgba(33, 147, 176, 0.6);
}
