6.1 理论讲解

什么是网页布局？

💡 老司机的比喻：

网页就像是一个“空房间”，HTML标签就是你买回来的“家具”。

布局，就是决定把沙发放在哪、电视挂在哪的技术！

  • 正常文档流：默认的摆放规则（从上到下，从左到右）。
  • 浮动 (Float)：让家具靠墙并排站。
  • 定位 (Position)：直接把家具钉死在指定的坐标上。

体验“正常文档流”

6.2 理论讲解

float 浮动：让元素飘起来

🎈 气球比喻：

给元素加上 float，它就像充满氢气的气球，脱离了地心引力，一直向左（或向右）飘。

直到碰到墙壁，或者碰到另一个浮动的气球才会停下。

最常用于：让原本上下排列的 div 能够并排显示（如图文混排）。

6.2 实操任务

让两个盒子并排牵手

任务：使用 float: left; 让左青龙、右白虎并列。

<style>
  .box {
    width: 200px; height: 100px;
    float: left; /* 核心魔法：向左浮动 */
  }
  .blue { background: lightblue; }
  .green { background: lightgreen; }
</style>

<div class="box blue">左青龙</div>
<div class="box green">右白虎</div>

浮动成功啦！

你会看到：原本上下排列的盒子，像排队一样紧紧挨在了一起。

⚠️ 警告：浮动很好用，但它会引发一个“大灾难”，我们稍后揭晓！

6.3 理论讲解

BFC：神秘的结界

BFC 全称是 Block Formatting Context (块级格式化上下文)。

🛡️ 魔法结界比喻：

你可以把 BFC 想象成一个封闭的“透明玻璃罩”。

不管玻璃罩里面的元素怎么翻江倒海（比如疯狂浮动），都绝不会影响到玻璃罩外面的排版！

6.3 实操任务

如何召唤 BFC 结界？

任务：给父容器添加特定的 CSS 属性来触发 BFC。

.container {
  /* 咒语1：溢出隐藏（最常用） */
  overflow: hidden; 
  
  /* 咒语2：设为行内块 */
  display: inline-block;

  /* 咒语3：绝对定位 */
  position: absolute;
}

记住最常用的咒语：overflow: hidden;，它能解决很多诡异的bug。

6.4.1 核心难点

浮动的灾难：高度塌陷

📦 纸箱比喻：

父元素是一个纸箱，里面装满了气球（子元素）。

突然，气球全 float 浮起来（飘出纸箱）了！

纸箱里面空了，啪！纸箱瘪了（父元素高度变成了0）。

后面的普通元素会钻到气球下面，页面排版彻底大乱！

6.4.2 实操任务

用 overflow: hidden 撑开纸箱

任务：利用刚才学的 BFC 结界，把浮动的子元素“关”在父元素里。

<style>
  /* 父元素（纸箱）加上这句咒语 */
  .parent {
    border: 5px solid black;
    overflow: hidden; /* 触发 BFC，自动包含浮动元素！ */
  }
  /* 子元素（气球）正在浮动 */
  .child { float: left; width: 100px; height: 100px; }
</style>
6.4.2 预期效果

结界生效！高度恢复

父元素变成了一个 BFC 结界，强制把内部的浮动元素计算在内。

浮动的子元素

注意：如果子元素需要故意溢出父元素显示，就不能用这招哦！

6.4.3 理论讲解

clear：划清界限

除了给父元素加结界，我们还可以让下面的元素“拒绝”浮动。

🧱 绝缘墙比喻：

clear 就是一堵绝缘墙。

对某个元素设置 clear: both，意思是：“我左右两边都不要有浮动的气球，我要自己另起一行！”

6.4.4 实操任务

把底部的页脚逼下去

任务：阻止 Footer 跑到上方浮动元素的旁边。

<style>
  .float-box { float: left; width: 50%; height: 80px; }
  .footer {
    clear: both; /* 核心：清除左右两侧的浮动 */
    background: #333; color: white;
  }
</style>
<div class="float-box">浮动区域</div>
<div class="footer">我是页脚，我要独立一行</div>
6.4.5 进阶理论

行业标杆：Clearfix 黑科技

overflow: hidden 有时候会切掉需要的溢出内容。

在同级元素加 clear: both 会增加无意义的 HTML 标签。

👻 幽灵墙比喻：

利用 CSS 伪元素 ::after，在父容器内部的最后面，悄悄塞入一堵看不见的、带有 clear: both 的墙。

6.4.5 实操任务

默写前端传家宝：Clearfix

任务：将这段代码加到你的 CSS 中，并赋予塌陷的父元素。

/* 将这个类名加给“瘪掉的父元素” */
.clearfix::after {
  content: "";       /* 1. 悄悄塞入一个空内容 */
  display: block;    /* 2. 把它变成块级元素（墙） */
  clear: both;       /* 3. 核心：清除所有浮动 */
}

这段代码请务必刻在你的键盘上！走到哪用到哪！

6.4.5 预期效果

纸箱又饱满啦！

给父元素添加 .clearfix 后，父元素被幽灵墙完美撑开，背后的灰色背景和红色边框包裹住了浮动内容。

6.8 理论讲解

相对定位 (relative)：影分身术

🥷 影分身比喻：

元素设置了 relative，就像施展了影分身。

它的“本体”（占用的物理空间）还留在原地不动，死死占着坑；

但它的“肉身”（视觉显示）可以根据 top/left 随意偏移。

6.8 实操任务

微调按钮的位置

任务：把按钮向下移动 15px，但不影响周围的排版。

<style>
  .btn {
    position: relative; /* 开启相对定位 */
    top: 15px;          /* 距离原来的顶部向下偏移 */
    left: 20px;         /* 距离原来的左侧向右偏移 */
  }
</style>
<button>普通按钮</button>
<button class="btn">相对定位按钮</button>
6.8 预期效果

位置偏移，但坑还在

红色按钮视觉上挪走了，但它原先的位置依然被保留着，没有被旁边的文字挤占。

普通按钮 相对定位按钮 ←这里是留下的坑
6.6 理论讲解

绝对定位 (absolute)：灵魂出窍

👻 灵魂出窍比喻：

一旦元素被设置为 position: absolute;，它就“脱离文档流”了，不占原来的空间。

它会飘在空中拼命往上找：“我祖宗里谁被定位过？”

找到后，就以那个祖宗为原点移动；如果都没定位，就以浏览器窗口为原点。

6.6 实操任务

在卡片右上角加个“新消息”红点

任务：使用绝对定位，把小红点钉在卡片右上角。

<style>
  /* 祖宗必须有定位，留住小红点 */
  .card { position: relative; width: 200px; }
  
  /* 小红点绝对定位 */
  .badge {
    position: absolute;
    top: -10px; right: -10px; /* 负值偏移 */
    background: red; color: white;
  }
</style>
6.6 预期效果

子绝父相，黄金搭档！

红色小徽章乖乖地贴在了卡片的右上角边缘，没有飞满全屏。

用户资料卡
99+

行业黑话：子绝对定位(absolute)，父相对定位(relative)，简称“子绝父相”。

6.10 理论讲解

固定定位 (fixed)：死死盯住你

📌 屏幕贴纸比喻：

fixed 就像直接贴在显示器屏幕上的一块狗皮膏药（贴纸）。

不管你怎么疯狂滚动鼠标滚轮，页面的内容在变，但它的位置永远锁定在屏幕上的同一个地方。

6.10 实操任务

制作“返回顶部”按钮

任务：把按钮锁定在浏览器屏幕的右下角。

<style>
  .go-top {
    position: fixed;   /* 核心魔法：固定定位 */
    bottom: 30px;      /* 永远距离屏幕底部 30px */
    right: 30px;       /* 永远距离屏幕右侧 30px */
    background: #2ecc71; color: white;
  }
</style>

6.7 理论讲解

z-index：谁在最上面？

既然定位会让元素飘在空中，那两个元素重叠了，谁盖住谁呢？

🍔 千层汉堡比喻：

屏幕不仅有左右 X 轴和上下 Y 轴，还有垂直于屏幕戳向你眼睛的 Z 轴。

z-index 的数值越大，元素就越靠近你的眼睛（在越上面的一层）。

6.7 实操任务

让底部的盒子翻身做主人

任务：利用 z-index 改变层叠顺序。

<style>
  /* 红盒子在下面，z-index 比较小 */
  .box1 { position: absolute; background: red; z-index: 1; }
  
  /* 蓝盒子在上面，z-index 比较大 */
  .box2 { 
    position: absolute; background: blue; 
    top: 30px; left: 30px; z-index: 99; 
  }
</style>
6.7 预期效果

成功突围！

因为 box2 的 z-index 是 99，蓝盒子稳稳压在红盒子上。

Box1 (z:1)
Box2 (z:99)

核心法则：z-index 只对开启了定位（非 static）的元素生效哦！

6.9 理论讲解

自适应宽度布局：像水一样

💧 杯子与水比喻：

不再使用死板的像素 (px)，而是使用百分比 (%)。

浏览器窗口像一个杯子，元素像水。杯子变大，水就铺开；杯子变小，水就收缩。

这是现代响应式网页（适配手机和电脑）的基础思维。

6.9 实操任务

7:3 比例的自适应双栏

任务：左边占70%，右边占30%，且永远并排。

<style>
  .left-side {
    float: left; width: 70%;  /* 写百分比！ */
    background: #e74c3c;
  }
  .right-side {
    float: left; width: 30%;  /* 刚好填满 100% */
    background: #f1c40f;
  }
</style>
6.9 预期效果

完美贴合的比例

不管预览框拉多长，它们始终保持 7:3 比例严丝合缝。

6.11 综合运用

终极挑战：搭建经典后台骨架

现在我们要把学到的招式融会贯通，搭建一个经典网页结构：

  • 🏠 Header (头部)：横跨全屏，在最上方。
  • 🚪 Sidebar (侧边栏)：固定宽度 200px，向左浮动。
  • 🛋️ Content (主内容)：在右侧自适应宽度填满剩余空间。
  • 🧱 Footer (底部)：在最下方，必须清除浮动。
第一步：HTML结构

打好网页的地基

严谨的嵌套关系是布局成功的关键：

<div class="header">我是头部</div>

<!-- 中间区域，包含浮动元素，必须清除浮动！ -->
<div class="clearfix">
  <div class="sidebar">左侧菜单 (200px)</div>
  <div class="content">右侧主要内容 (自适应)</div>
</div>

<div class="footer">我是底部</div>
第二步：CSS 侧边栏配置

让菜单靠左站好

给左侧栏加上宽度并浮动：

/* 头部简单设个高度和背景色即可 */
.header { height: 60px; background: #34495e; color: white;}

/* 侧边栏：核心是 float 和 定死宽度 */
.sidebar {
  float: left;     /* 1. 向左浮动 */
  width: 200px;    /* 2. 宽度写死 200px */
  height: 400px;   /* 模拟内容高度 */
  background: #bdc3c7;
}
第三步：CSS 主内容自适应

巧妙让出左侧空间

右侧不需要浮动，只需用 margin 避开左边：

/* 右侧主内容 */
.content {
  /* 核心技巧：左外边距大于等于侧边栏宽度 */
  /* 这样内容就不会被浮动的侧边栏遮挡 */
  margin-left: 210px; 
  height: 400px;
  background: #ecf0f1;
}

因为 div 是块级元素，默认占满剩余宽度，完美实现一侧固定一侧自适应！

第四步：CSS 底部收尾

守护底线，坚决清除浮动

为了防止侧边栏太长戳破中间区域，底部必须自保：

/* 底部区域 */
.footer {
  /* 最关键的一句，绝不和浮动元素做邻居 */
  clear: both;     
  height: 80px;
  background: #2c3e50;
  color: white;
}
终极效果展示

经典的后台布局骨架大功告成！

完全遵循刚才我们敲下的 CSS 逻辑渲染出的真实模样：

.header (横跨全屏)
.sidebar
(向左浮动，定宽)
.content
(利用 margin-left 避让，宽度自适应)
.footer (clear: both 清除浮动)
6.12 课程小结

今天我们学到了什么？

  • float 浮动：元素并排利器，但别忘了处理高度塌陷。
  • BFC & Clearfix：清除浮动、撑开父元素的内功心法和外功招式。
  • absolute 绝对：灵魂出窍，寻找定位祖宗（子绝父相）。
  • relative 相对：影分身占坑，做祖宗的最佳人选。
  • fixed 固定：狗皮膏药，死盯浏览器可视窗口。
