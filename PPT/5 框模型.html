5.1 万物皆“盒子”

在前端开发者的眼里，网页并不是平面的画纸，而是由无数个看不见的盒子堆砌而成的乐高玩具。

生活中的比喻：

就像你去逛超市，货架上的商品。不管里面的东西是圆的（苹果）、软的（毛巾）还是奇形怪状的，厂家都会给它套上一个方方正正的包装盒，方便摆放。

网页里的标签（如 <div>, <p>, <img>），在浏览器眼里，统统都是一个矩形的“盒子”！

5.2 框模型的要素

解剖一个标准的前端“快递箱”

7

5.2 框模型的四大要素

每一个 HTML 盒子，都由四个部分组成。闭上眼睛，想象你寄了一个易碎品（比如玻璃杯）的快递：

  • Content (内容)：玻璃杯本身（文字、图片）。
  • Padding (内边距)：杯子和纸箱之间的防震泡沫纸。
  • Border (边框)：快递纸箱的硬纸板。
  • Margin (外边距)：这个纸箱和旁边纸箱之间的安全距离（防碰撞）。
8

Padding写法合集

padding:10px;  1个值：上下左右内边距padding都为10px

padding:10px 20px;  2个值：上下10px,左右20px

padding:25px 50px 75px;  2个值：上25px,左右50px,下75px

padding:25px 50px 75px 100px;  2个值：上25px,右50px,下75px,左100px

9

Padding写法合集

padding-top:10px;  仅有上内边距padding为10px

padding-right:20px;  仅有右内边距padding为20px

padding-bottom:15px;  仅有下内边距padding为15px

padding-left:10px;  仅有左内边距padding为10px

9

实操任务 2.1：塞入防震泡沫 (Padding)

任务目标：让文字和盒子的边缘保持距离，不要紧贴着。

<div class="box-step1">我是玻璃杯（内容）</div>

<style>
  .box-step1 {
    background-color: lightblue;
    /* padding: 上 右 下 左; */
    padding: 30px; /* 塞入 30px 厚的泡沫纸 */
  }
</style>

任务 2.1：预期效果

没有 padding 时，文字紧贴边缘：

加了 padding: 30px; 后，内容被保护起来了，盒子被“撑大”了：

我是玻璃杯（内容）

5.3 盒子的宽度和高度

前端界的一大“骗局”：到底多大才算大？

15

5.3 灵魂拷问：100px 到底有多宽？

如果你在 CSS 中写了 width: 100px;，这个盒子在屏幕上占据的宽度真的是 100px 吗？

错！（这就是无数新手熬夜掉头发的原因 😂）

在标准框模型中，你设置的 width 和 height，仅仅只是玻璃杯（Content）的尺寸！

真实占地宽度 = 内容宽度width + 左 padding + 右 padding + 左 border + 右 border

16

实操任务 3.1：算算真实体型

任务目标：写一个盒子，看看它到底被撑得有多大。

<div class="math-box">我胖吗？</div>

<style>
  .math-box {
    width: 200px;
    padding: 20px;
    border: 5px solid black;
    background-color: lightgreen;
  }
</style>
17

任务 3.1：预期效果与计算

虽然你写了 width: 200px，但在浏览器里审查元素时，它的实际宽度是：

200(内容) + 20(左内边距) + 20(右内边距) + 5(左边框) + 5(右边框) = 250px！

生活比喻：你买了一张 2米 宽的床垫（内容），但你加了 20厘米的床框（Padding/Border），结果你的卧室放不下了！

18

CSS 救星降临：border-box

每次都要做算术题太折磨人了！怎么才能让我设置 width: 200px，它最终的占地就是 200px 呢？

我们要改变规则，使用：box-sizing: border-box;

买房比喻：

  • 默认模式 (content-box)：买房看“套内面积”，加上公摊后总价超出预算。
  • 神仙模式 (border-box)：买房看“建筑面积”，设置多大就是多大，边框和内边距会向内挤压内容空间！
19

实操任务 3.2：启用魔法盒子

任务目标：使用 border-box 锁定盒子的最终大小。

<style>
  .magic-sizing {
    /* 划重点！现代前端必写的属性 */
    box-sizing: border-box; 
    
    width: 200px;
    padding: 20px;
    border: 5px solid black;
    background-color: pink;
  }
</style>
<div class="magic-sizing">我被锁定了！</div>
20

任务 3.2：预期效果

加上 box-sizing: border-box; 后：

不管你把 padding 和 border 设置多大，盒子的总宽度死死锁定在 200px！

浏览器会自动帮你做减法，压缩里面文字（内容）的活动空间。

老司机的忠告：在实际开发中，我们通常会在最开头对所有元素加上全局设定 `* { box-sizing: border-box; }`

5.4 框模型的应用

如何优雅地摆弄这些盒子

22

应用 1：让盒子“C位出道” (水平居中)

我们在网页上经常看到居中对齐的内容块。怎么让一个盒子在父元素里水平居中呢？

秘诀：自动分配外边距！

当我们把盒子的左右 margin 设置为 auto 时，浏览器就会平分两边的剩余空间，盒子就被推到了正中间！

margin: 0 auto;

23

实操任务 4.1：盒子居中术

任务目标：让一个 400px 宽的盒子在页面中水平居中。

<div class="center-box">我是C位盒子</div>

<style>
  .center-box {
    width: 400px;
    background-color: orange;
    /* 上下边距为0，左右为自动平分！ */
    margin: 0 auto; 
    text-align: center; /* 让文字也居中 */
  }
</style>
24

任务 4.1：预期效果

盒子乖乖地跑到了页面的正中央：

我是C位盒子

注意前提条件：盒子必须是有明确宽度的块级元素（设置了具体的 width），margin: 0 auto; 才会生效哦！

25

经典陷阱：外边距塌陷 (Margin Collapse)

这又是前端的一个“玄学”设定。想象一下：

上面盒子的 margin-bottom 是 30px。

下面盒子的 margin-top 是 20px。

你以为它们之间的距离是 30 + 20 = 50px 吗？

错！它们会“打架”，距离不是相加，而是“谁大听谁的”，最终距离是 30px！

26

实操任务 4.2：避开塌陷陷阱

任务目标：体会并解决外边距塌陷。

<style>
  .boxA { margin-bottom: 50px; background: #ddd;}
  .boxB { margin-top: 50px; background: #eee;}
</style>
<div class="boxA">上面的盒子</div>
<div class="boxB">下面的盒子</div>

老师支招：怎么避免脑子糊涂？最简单的实战技巧是：**统一只设置单方向的外边距**（比如大家统一只设置 `margin-bottom`），不让它们有发生重叠的机会！

27

用边框画三角

通过设置宽高为0，并利用边框的交界处，可以画出各种三角形。

<style>
  .triangle {
    width: 0;
    height: 0;
    border: 100px solid transparent;
    border-bottom-color: red;
  }
</style>
<div class="triangle"></div>

效果：一个向上的红色三角形（底部边框为底边，其他三边透明）。

5.5 display 属性

给盒子分配不同的“性格”

28

5.5 盒子的三种基本性格

在 HTML 中，盒子生来性格不同，主要由 display 属性决定：

  • block (块级)：霸道总裁。 必须独占一行，宽、高、内外边距随便设。（代表：<div>, <p>, <h1>）
  • inline (行内)：随和群演。 大家挤在同一行，设置宽高无效！靠内容撑开。（代表：<span>, <a>）
  • inline-block (行内块)：混血儿。 既能排在同一行，又能随意设置宽高！

任务目标：测试不同标签天生的 display 属性。

<div style="background: pink;">我是div2，我也独占一行</div>

随和群演 <span>：并排站立。

实操任务 5.2：性格改造 (inline-block)

任务目标：我们想要多个方块并排显示，但又要能设置宽度和高度。怎么办？

<style>
  .nav-item {
    display: inline-block; /* 划重点！转换性格 */
    width: 100px;

    color: white;

5.5 终极魔法：隐身衣 display: none;

有时候我们需要把某个盒子彻底从网页上抹去（比如关掉弹窗）。

此时，只需使用 display: none;。

特点：

  • 盒子完全消失，不占用任何网页空间。
  • 后边的元素会直接顶上来。
  • 这是配合 JavaScript 做交互特效（如显示/隐藏菜单）时最常用的 CSS 属性！

5.6 小结

收下这份前端武功秘籍

35

框模型核心秘籍复盘

  1. 记住解剖图：一个盒子 = 内容(Content) + 内边距(Padding) + 边框(Border) + 外边距(Margin)。
  2. 避开尺寸陷阱：养成好习惯，遇到尺寸算不清，直接祭出 box-sizing: border-box;。
  3. 居中法宝：固定宽度的块级盒子，用 margin: 0 auto; 稳稳 C位。
  4. 搞懂性格： block 独占一行，inline 挤在一起且不能设宽高，inline-block 完美结合体。
