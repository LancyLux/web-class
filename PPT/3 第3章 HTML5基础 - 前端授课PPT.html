3.1 HTML 的发展 (打个比方)

学习代码就像是学盖房子。

  • 早期 HTML： 就像用泥巴糊墙。大家各显神通，代码乱七八糟，浏览器只能“猜”你想显示什么，房子摇摇欲坠。
  • XHTML 时代： 泥瓦匠被教导主任骂了，开始要求绝对的横平竖直。
  • HTML5 时代： 终于有了现代化预制板！高效、酷炫、包容性强！

3.1.1 XHTML：严厉的教导主任

XHTML 曾经统治过一段时期，它的特点是极致的严格：

  • 标签必须小写：写成 <HTML> 会被报错！
  • 标签必须闭合：少写一个 </p>，网页直接崩溃！
  • 属性必须加引号：id=box 不行，必须是 id="box"。

现状： 虽然现在不常用，但它给我们留下了“写规范代码”的好习惯。

3.1.2 HTML5：宽容的现代建筑师

终于，HTML5 诞生了！它就像一位酷酷的现代建筑师：

  • 包容： 标签大写？没闭合？它都能尽量帮你修复渲染。
  • 简洁： 文档声明从原来长长的一大串，变成了简单的 <!DOCTYPE html>。
  • 强大： 自带视频播放、画布绘图、本地存储等超强能力，不再依赖 Flash 这种插件！

3.2 HTML5 布局元素：为何需要语义化？

以前我们布局，满屏幕都是 <div>。这就好比你搬家，所有的箱子全长一样，完全不知道里面装的是衣服还是零食。

HTML5 引入了语义化标签，就是给你的纸箱子贴上清晰的标签！

  • 搜索引擎（如百度、谷歌）看得懂。
  • 盲人阅读器读得懂。
  • 我们程序员自己接手代码时，也不会抓狂了。

3.2 认识这些“带标签的箱子”

  • <header>：页头（网页的帽子/招牌）
  • <nav>：导航（网站的路牌，存放菜单链接）
  • <main>：主要内容（网页的身体，核心内容）
  • <article>：独立文章（可以单独成篇的一篇博客或新闻）
  • <section>：章节区块（将页面划分为不同功能区）
  • <footer>：页脚（网页的鞋子/版权声明）

【实操任务 1】搭建你的第一个 HTML5 别墅骨架

任务要求： 使用 HTML5 布局标签，搭建一个包含头部、导航、主体和页脚的简单博客页面骨架。

<body>
  <header>
    <h1>我的个人博客</h1>
  </header>

  <nav>
    <a href="#">首页</a> | <a href="#">关于我</a>
  </nav>

  <main>
    <section>这里是核心内容区</section>
  </main>

  <footer>版权所有 © 2026</footer>
</body>

【结果揭晓 1】别墅骨架展示

页面上会按顺序依次显示：

我的个人博客

首页 | 关于我

这里是核心内容区

版权所有 © 2026

虽然现在还没有 CSS 来美化，但骨架非常健康，极具专业范儿！

3.3 常用 HTML5 元素：做笔记的荧光笔

我们在读书时，总喜欢用笔画重点。HTML 也有这样的“画笔”：

  • <mark> 标签：高亮显示文本。（就像黄色的荧光笔）
  • <strong> 标签：文字加粗，表示强调。
  • <em> 标签：文字斜体，表示语气上的强调。

比单纯使用 CSS 改变颜色更好，因为它们带有语义！

【实操任务 2】给你的文章画重点

  <strong>HTML</strong>、
  <strong>CSS</strong> 和 
  <strong>JavaScript</strong>。
  其中，<mark>多动手敲代码</mark>是学好前端的唯一捷径！
</p>

【结果揭晓 2】自带高亮特效

学习 Web 前端开发，最基础的三剑客是： HTML、 CSS 和 JavaScript。 其中，多动手敲代码是学好前端的唯一捷径！

浏览器默认会为 <mark> 标签加上黄色的背景底色，非常醒目！

3.3 常用 HTML5 元素：加载条与仪表盘

以前要做个游戏血条或者加载进度，要写一堆复杂的 CSS 和 JavaScript。现在 HTML5 给了你现成的组件：

  • <progress> 标签：进度条。表示一个任务完成的程度，比如下载进度。
  • <meter> 标签：度量衡 / 仪表盘。表示已知范围内的数值，比如手机电量、磁盘空间。

【实操任务 3】制作一个下载进度条和磁盘容量盘

任务要求： 直接使用 HTML 标签展示文件下载到了 70%，以及 C 盘空间使用了 90%。

<h3>文件下载进度：</h3>
<!-- value是当前值，max是最大值 -->
<progress value="70" max="100"></progress> 70%

<h3>C盘空间使用情况：</h3>
<!-- 可以设置高低风险值，当达到90时通常会变红 -->
<meter value="90" min="0" max="100" low="30" high="80">
</meter> 90%

【结果揭晓 3】不写一行 CSS 的进度组件

文件下载进度：
70%

C盘空间使用情况：
90% (极高，通常渲染为红色或黄色)

不同的浏览器会自带不同的默认 UI 样式，但功能完全生效！

3.4 HTML5 属性：给标签赋予超能力

HTML5 不仅带来了新标签，还带来了全局属性，它们可以加在几乎任何标签上，就像魔法一样：

  • hidden 属性：隐身斗篷。加上它，元素就在页面上消失了。
  • contenteditable 属性：点石成金的魔法棒。加上它，网页上的文字就能像 Word 文档一样被用户直接修改！

【实操任务 4】体验隐身斗篷与魔法棒

任务要求： 编写一段可以被用户直接编辑的签名，以及一个被隐藏的秘密彩蛋。

<!-- 添加 contenteditable="true" 让文字可编辑 -->
<p contenteditable="true">
  这是一段神奇的文字，你用鼠标点一下，就可以直接修改我啦！
</p>

<!-- 添加 hidden 属性隐藏元素 -->
<div hidden>
  这是一个隐藏的宝藏，普通用户在页面上是看不见我的！
</div>

【结果揭晓 4】网页变身记事本

这是一段神奇的文字，你用鼠标点一下，就可以直接修改我啦！

(此时隐藏的宝藏 div 完全不可见，不占据任何空间)

contenteditable 属性是开发“网页富文本编辑器”的核心基石！

3.4 HTML5 属性：data-* 自定义属性

有时候，我们需要在网页元素上偷偷塞一点数据，方便以后用 JavaScript 获取。

这就是 data-* 属性，它就像是在你的衣服内侧缝了一个隐秘的口袋。

  • 规则：以 data- 开头，后面跟着你自定义的名字。
  • 示例：data-user-id="1001"
  • 好处：规范化存储私有数据，且不会影响页面的任何显示。

【实操任务 5】给按钮缝上隐秘口袋

任务要求： 制作一个商品购买按钮，并将商品的 ID 和价格隐藏在标签的 data 属性中。

<!-- 给按钮添加 data-id 和 data-price 属性 -->
<button 
    data-id="apple_001" 
    data-price="9.9"
>
  购买苹果
</button>

【结果揭晓 5】前端工程师的“暗号”

页面上只显示普通的按钮。但后续如果你学习了 JavaScript，只需一行代码就能读取到 9.9 这个价格去结算，非常巧妙！

3.5 HTML5 表单：会思考的智能助手

以前网页里的表单（输入框）非常笨，只能让你敲字。

如果要验证你输入的是不是邮箱，或者让你选个颜色、选个日期，程序员要写几百行 JavaScript 代码才能实现。

HTML5 表单迎来了史诗级加强！它们现在自带日历、调色板和智能提示了！

3.5.1 datalist 元素：智能下拉提示

你在百度搜索时，打一个字，下面就会弹出一排建议对吧？

现在我们用 HTML5 的 <datalist> 标签，几行代码就能实现类似的效果！

  • 它是一组隐藏的备选项。
  • 必须把它和 <input> 框通过 id 绑定起来才能召唤它！

【实操任务 6】制作一个自带提示的浏览器输入框

任务要求： 创建一个文本输入框，当用户输入内容或双击时，提示 Chrome、Edge 等浏览器名称。

<!-- 1. input输入框，用 list 属性绑定 datalist 的 id -->
<input type="text" list="browsers" placeholder="选择浏览器">

<!-- 2. datalist 提供数据选项，通过 id 关联 -->
<datalist id="browsers">
  <option value="Chrome">
  <option value="Edge">
  <option value="Firefox">
  <option value="Safari">
</datalist>

3.5.2 autocomplete 属性：浏览器的记忆力

你有没有发现，每次在购物网站填收货地址，浏览器经常会询问你：“要帮你自动填充以前写过的地址吗？”

这就是 autocomplete (自动完成) 属性的功劳。

  • autocomplete="on"：开启历史记录记忆（默认）。
  • autocomplete="off"：关闭记忆。比如涉及到银行密码、验证码时，我们必须把它关掉以保证安全！

3.5.3 酷炫的新 input 类型

只要修改 <input> 的 type 属性，就能召唤各种神奇控件：

  • type="color"：弹出一个原生的系统调色板！
  • type="date"：弹出一个原生的日历让你选日期！
  • type="range"：变成一个可以左右拖动的滑块！
  • type="email" / type="url"：在手机端会呼出带有 "@" 或 ".com" 专属键盘！

3.6 音频和视频：告别 Flash 的黑暗时代

你们可能不知道，以前在网页上看个视频，电脑里必须得安装一个叫 Flash Player 的插件，它又卡又容易崩溃。

HTML5 出现后，霸气地说：“插件时代结束了，视频和音频，我自己原生支持！”

这就好比买手机自带了高清相机，再也不用外接个笨重的镜头了。

3.6 认识 audio 和 video

  • <audio> 标签：用于播放音乐/音频。
  • <video> 标签：用于播放电影/视频。

关键属性：

  • src：文件路径（歌曲或视频在哪里）。
  • controls：魔法开关！加上它，浏览器就会自动显示出播放、暂停、音量调节按钮！

【实操任务 8】在网页里开一个微型影院

任务要求： 在页面中嵌入一段音乐（带控制条）和一段视频（设置宽高并带控制条）。

<!-- 插入音频，加上 controls 显示播放按钮 -->
<h3>背景音乐：</h3>
<audio src="music.mp3" controls></audio>

<!-- 插入视频，设置宽为400像素 -->
<h3>猫咪视频：</h3>
<video src="cat.mp4" controls width="400">
  你的浏览器太老啦，不支持视频播放。
</video>
